3 \section OverviewIntro What is the Stack-Of-Tasks ?
4 
5 The goal of the Stack-of-Tasks (SoT) is to provide a SDK in order to:
6  * Write control problem for complex robots such as humanoid robots
7  * Test them quickly using geometric settings (rviz, gepetto-viewer), dynamic simulators (ODE,Bullet), system simulator (Gazebo)
8  * Deploy them in real robots (TALOS, HRP-2)
9 Depending on your target you might not be interested in using the full set of software we are providing. But the main goal of this SDK is to consider a robot model
10 \\( RM \\), its current state \\( x \\) and computes a control vector \\(u \\) to make the robot performs actions.
11 
12 All our controllers are defined as optimization problems. We are using either off-the-shelf solvers or our own solvers, depending of the targeted applications. The free variables are the control vector \\( u \\) to apply to the system. They are constraints which can be of several kinds: the mechanical constraints of the system with the environment, the behavior that we want the robot to follow. A short introduction on how those two constraints can be mathematically formulated can be found
13 in \ref b_controllers_information . We strongly recommend that you read it if you do not have a good grasp of robot control.

30 \section OrganizationIntro Organization
31 
32 The SoT SDK is organized in packages in the following modules:
33  * Core algorithmic libraries: They all can be used *WITHOUT* the other modules
34  * pinocchio: A fast and flexible rigid body dynamic library implementing proper algebraic computation of mechanical quantities.
35  * tsid: A Task Space Inverse Dynamics library.
36  * jrl-walkgen: A library targed to make humanoid robot walks.
37  * Modules to handle a dynamic graph of computational entities:
38  * dynamic-graph: A C++ graph library allowing to build a graph of computational entities. This is not a middleware, it handles time dependencies.
39  * dynamic-graph-python: Python bindings to manage the dynamic-graph structure. Plus a C embedded python interpreter which can be used together with the C++ library for on-line graph modification
40  * Solvers to generate control laws:
41  * sot-core: Kinematics based solver. It is strongly based on the stack-of-taks paradigm and exposed several objects. This is has tested on several occasions and platforms and is allowing fast prototyping of robot motions.
42  * sot-dynamic-pinocchio: A sot-wrapper around pinocchio. This library exposes all the robot quantities (end-effectors position, CoM, mass matrix)
43  * sot-torque-control: A sot-wrapper around TSID.
44  * sot-pattern-generator: A sot-wrapper around jrl-walkgen. This library provides feet, waist, ZMP and CoM trajectories.
